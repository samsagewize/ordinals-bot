What You Need First

Required
  • Node.js 18+ — download at nodejs.org
  • A server or computer to host the bot 24/7 (Railway, Render, DigitalOcean, or your own VPS)
  • Your Ordinookis Discord server — you need Admin permissions
  • A Bitcoin wallet address you control — this is where the $5 fees go
2

Create the Discord Bot

Required

On Discord Developer Portal

  1. Go to discord.com/developers/applications
  2. Click New Application → Name it Ordinookis Bot
  3. Go to the Bot tab on the left sidebar
  4. Click Add Bot → confirm
  5. Under Privileged Gateway Intents, enable:
    SERVER MEMBERS INTENT ✅
  6. Click Reset Token → copy and save it somewhere safe
  7. Go to General Information → copy the Application ID

Invite the Bot to Your Server

  1. Go to OAuth2 → URL Generator
  2. Under Scopes, check: bot and applications.commands
  3. Under Bot Permissions, check:
    • ✅ Manage Roles
    • ✅ Send Messages
    • ✅ Use Slash Commands
    • ✅ Read Message History
  4. Copy the generated URL at the bottom → open it in your browser
  5. Select your Ordinookis Discord server → Authorize

Get Your Server (Guild) ID

  1. In Discord, go to Settings → Advanced → enable Developer Mode
  2. Right-click your server icon in the sidebar
  3. Click Copy Server ID

Install & Configure the Bot

Required

Install Dependencies

# Clone or download the bot files, then:
npm install

# Copy the example env file
cp .env.example .env

Fill in Your .env File

Open .env in any text editor and fill in these values:

VariableWhat it isWhere to get it
DISCORD_BOT_TOKEN Your secret bot token Bot tab → Reset Token
DISCORD_CLIENT_ID Your app's ID General Information → Application ID
DISCORD_GUILD_ID Your Discord server ID Right-click server → Copy Server ID
PAYMENT_BTC_ADDRESS Your BTC wallet address to receive fees Your Xverse / Leather / hardware wallet
PAYMENT_SATS Fee in satoshis (5000 ≈ $5) Leave as 5000
HOLDER_ROLE_NAME Name of role to give verified holders Leave as Ordinookis Holder
INSCRIPTION_IDS Comma-separated inscription IDs (optional but recommended) See Step 4 below
⚠️ Never share your DISCORD_BOT_TOKEN — treat it like a password. If you accidentally post it publicly, go to the Developer Portal and reset it immediately.
4

Configure Ordinookis Collection

Recommended

Option A — Add Inscription IDs (most accurate)

This is the most reliable method. Go to ordiscan.com/collections or satflow.com/ordinals/ordinookis and find the Ordinookis collection. Get the inscription IDs and add them to your .env:

INSCRIPTION_IDS=abc123txidherei0,def456txidherej0,...

Option B — Collection Name Matching (automatic fallback)

If you leave INSCRIPTION_IDS blank, the bot automatically searches for "ordinookis" in the collection name field across Hiro, Ord.io, and Ordinals.com. This works as long as your collection is indexed by those APIs.

💡 Pro tip: You can get your collection's inscription IDs from the Ordinookis listings on Satflow. Click any item → the URL contains its inscription ID (e.g. abc123...i0).

Set Up Your Discord Server

Required

Role Hierarchy — CRITICAL

The bot can only assign roles that are below its own role in the hierarchy. If this is wrong, role assignment will silently fail.

  1. Go to your Discord server → Server Settings → Roles
  2. Find the Ordinookis Bot role (auto-created when you invited it)
  3. Drag it above the Ordinookis Holder role
  4. Save changes
Correct order (top = highest):
🔴 Admin
🟠 Moderator
🟡 Ordinookis Bot ← must be here
🟢 Ordinookis Holder ← bot assigns this
⚪ @everyone

Create a #verification Channel

  1. Create a new channel called #verify-here or #get-role
  2. Pin a message explaining the steps (see below)
  3. Optionally restrict the channel so only unverified users see it

📌 Suggested pinned message:

👋 Welcome, Ordinookis holder!

To get your exclusive holder role:
1️⃣ Run /verify <your_btc_address>
2️⃣ Send the one-time $5 fee in BTC to the address shown
3️⃣ Run /confirm <your_txid> after sending
4️⃣ Role is assigned automatically once confirmed on-chain ✅

Check your progress anytime with /status

Bot Commands Reference

Reference
/verify <btc_address>
👤 User command

Links wallet & shows payment instructions with your BTC address + fee amount.

/confirm <txid>
👤 User command

Submits payment TX. Bot verifies on-chain, checks Ordinals, grants role.

/status
👤 User command

Shows current verification progress (wallet linked → paid → verified → role).

/admin lookup <user>
🛡️ Mod only

Shows full verification record for any Discord user.

/admin manual_verify <user>
🛡️ Mod only

Manually grant role to a user (bypasses payment/ordinal check).

/admin stats
🛡️ Mod only

Shows total linked, paid, verified, and role-granted counts.

What Your Users Experience

Reference
💬

User runs /verify bc1q...their_address

Bot saves their wallet address and sends them a DM with your BTC payment address, the exact amount to send (5000 sats), and instructions to run /confirm after.

₿

User sends 5000 sats from their Ordinookis wallet

They use Xverse, Leather, or any BTC wallet. Sending FROM the same address proves wallet ownership — no signature needed.

💬

User runs /confirm <txid>

They grab the transaction ID from their wallet's history and submit it. Bot looks it up on mempool.space instantly.

🔍

Bot verifies payment + Ordinookis holding in parallel

Checks mempool.space for confirmed TX. Simultaneously queries Hiro API, Ord.io, and Ordinals.com for Ordinookis in that wallet. Any one API confirming = pass.

🎉

Role automatically assigned

User gets the Ordinookis Holder role instantly. Bot replies with confirmation showing inscription ID and which API confirmed it.

Pre-Launch Checklist

Do this before going live
  • Discord bot created, token copied
  • Bot invited to server with Manage Roles + Slash Commands permissions
  • Developer Mode enabled, Guild ID copied
  • .env file filled in with all required values
  • Your BTC payment address set in PAYMENT_BTC_ADDRESS
  • npm run deploy run successfully — commands visible in Discord
  • Bot role is above Ordinookis Holder in role hierarchy
  • #verify-here channel created with pinned instructions
  • Test with your own wallet: run /verify → pay → /confirm → get role
  • Bot running 24/7 on a server (Railway / Render / VPS)
